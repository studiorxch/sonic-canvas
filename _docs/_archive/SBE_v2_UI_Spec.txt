SBE v2 — UI REFACTOR SPEC (CANVAS-FIRST)

OBJECTIVE
Refactor UI from a 3-column layout into a canvas-first system with floating controls and contextual inspector.

Goal:
- maximize canvas space
- remove excess UI
- show controls only when needed

DESIGN PRINCIPLES

1. Canvas is always visible and centered
2. No permanent side panels
3. Inspector appears only when something is selected
4. Controls float, they do not consume layout space
5. No duplicate controls
6. Keyboard-first workflow

REMOVE EXISTING UI

Delete:
- Left panel (tools, transport, canvas controls, scene, MIDI, swarm)
- Right panel (persistent inspector)
- Grid layout (.app-shell with columns)

NEW STRUCTURE

<main class="stage-shell">
  <div class="canvas-stage">
    <div class="canvas-wrap">
      <canvas id="engine-canvas"></canvas>
    </div>
  </div>
</main>

INSPECTOR BEHAVIOR

Show inspector only when an object is selected.

TOOL STATE

Toolbar reflects active tool.

KEYBOARD SHORTCUTS

V = select
L = line
T = text
B = ball

Cmd+D = duplicate
Cmd+G = group
Shift+Cmd+G = ungroup
Delete = delete
Cmd+Z = undo

Tab = presentation mode
? = shortcuts

PRESENTATION MODE

Tab toggles UI visibility.

CANVAS SIZING

function updateCanvasAspect() {
  const wrap = document.querySelector(".canvas-wrap");
  wrap.style.aspectRatio = state.canvas.width + " / " + state.canvas.height;
