# SBE Glyph System v1 + Glyph Alphabet v1

## Overview

This document defines the **SBE Glyph System**, a monoline, segmented, motion-driven visual language used for generating sound.

These glyphs are NOT fonts.

They are:
- structural paths
- motion constraints
- rhythm generators

---

## Core Principle

form → motion → collision → sound → loop

---

## SBE Glyph System v1 Rules

### 1. Monoline Only
- Single stroke weight
- No thickness variation

---

### 2. Segmented Geometry
- No perfect curves
- Curves are broken into segments (2–4 lines)

---

### 3. Angular Bias
Preferred angles:
- 45°
- 90°
- Slightly imperfect diagonals

---

### 4. Continuous or Near-Continuous Paths
- Minimal pen lifts
- Maintain flow across the shape

---

### 5. Imperfect Symmetry
- Avoid perfect circles
- Allow slight distortion

---

### 6. Readable but Not Correct

Recognizable > Accurate

---

## Primitive Elements

All glyphs are constructed from:

- Horizontal line  —
- Vertical line    |
- Diagonal lines   /  \
- Corners          ⌐ ⌞ ∟
- Broken arcs      (approximated with segments)

## Non-Letter Glyphs (Core Library)

These are equally important as letters.

- Loop (segmented circle)
- Double Loop
- Zig Path
- Broken Square
- Cross (+ / X hybrid)
- Offset Rectangle
- Spiral (segmented)

---

## Usage in SBE

Glyphs act as:

- rhythm templates
- collision guides
- motion constraints

Each segment:
- influences timing
- affects collision spacing
- produces sonic variation

## User Flow

1. Select glyph
2. Place on canvas
3. Rotate / scale
4. Assign note (auto color)
5. Add motion (balls / swarm)
6. Play and record

---

## System Constraint (IMPORTANT)

Glyphs must follow system constraints, not typographic correctness.
